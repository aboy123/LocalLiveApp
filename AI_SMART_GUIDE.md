### 步骤2：获取API密钥

以OpenAI为例：
1. 注册 OpenAI 账号
2. 进入 https://platform.openai.com/api-keys
3. 创建新的API密钥
4. 复制密钥（格式：sk-xxxxxxxxxxxxx）

### 步骤3：修改配置

打开 `src/data/aiChatData.js`，找到 `aiApiConfig` 部分：

```javascript
export const aiApiConfig = {
  // 设置为 true 启用外部API
  useExternalApi: true,  // ← 改为 true
  
  // API类型: 'openai' | 'qwen' | 'custom'
  apiType: 'openai',     // ← 选择你的API类型
  
  // API密钥（替换为你的实际密钥）
  apiKey: 'sk-your-api-key-here',  // ← 填入你的密钥
  
  // API端点（通常不需要改）
  apiUrl: 'https://api.openai.com/v1/chat/completions',
  
  // 模型名称
  model: 'gpt-3.5-turbo',
};
```

### 步骤4：测试

重启开发服务器，发送消息测试：
```bash
npm run dev
```

## ⚠️ 注意事项

### 安全性提醒
1. **不要将API密钥提交到Git仓库**
2. 建议使用环境变量存储密钥

### 使用环境变量（推荐）

创建 `.env` 文件：
```env
VITE_AI_API_KEY=sk-your-api-key-here
VITE_AI_API_TYPE=openai
VITE_USE_EXTERNAL_AI=true
```

修改代码读取环境变量：
```javascript
export const aiApiConfig = {
  useExternalApi: import.meta.env.VITE_USE_EXTERNAL_AI === 'true',
  apiType: import.meta.env.VITE_AI_API_TYPE || 'openai',
  apiKey: import.meta.env.VITE_AI_API_KEY || '',
  apiUrl: 'https://api.openai.com/v1/chat/completions',
  model: 'gpt-3.5-turbo',
};
```

## 🐛 常见问题

### Q: 为什么只有一个AI助手了？
A: 为了简化用户体验，我们将4个角色整合成一个全能AI助手。它能自动识别你的需求，提供相应的帮助。

### Q: 本地回复够智能吗？
A: 本地回复系统已经相当智能，能识别情绪和话题，提供有针对性的回复。但如果需要真正的AI智能，可以接入外部API。

### Q: API调用失败怎么办？
A: 系统会自动降级到本地回复，不影响使用。

### Q: 如何测试API是否配置成功？
A: 发送一条消息，如果回复非常个性化且上下文连贯，说明API工作正常。

### Q: 可以自定义回复模板吗？
A: 可以！在 `getSmartAssistantTemplates` 方法中添加你自己的模板。
